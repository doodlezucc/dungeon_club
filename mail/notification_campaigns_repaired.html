Some details on what happened:
An unhandled error in the server code caused the periodic auto-save to stop auto-saving the account database. The same database is also used to store campaign ownership.
At the start of May, the server was forcefully shut down and restarted by what seems to have been a denial of service attack. Due to the last backup dating back to March 6th, all account related changes between the auto-save and the shutdown had been undone. To be clear, this "memory gap" did not revert the actual contents of any campaign, only their individual assigned owners.
By running a bunch of simulations on the pseudonymized server logs and by making a few educated guesses, relationships between accounts and created campaigns affected by the memory gap could eventually be established and were restored in the database.
If you find any campaigns still missing from your account, please don't hesitate to reach out by replying to this email or joining the community Discord server.
Sorry for any inconveniences the bug might have caused.
